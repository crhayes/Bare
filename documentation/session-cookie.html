Introduction

The session and cookie helpers, found in libraries/session/session.php and libraries/cookie/cookie.php respectively, provide an easy way to add encryption to sessions and cookies.

Sessions and cookies are encrypted with AES before being stored, which greatly improves their security.

Sessions

Start a Session

Starting a session is simple, and you can set an encryption key for increased security.

Session::start()->setKey('thisIsASecurityKeyForTheEncryption');

Handling Sessions

// Set a session
Session::set('username', $user->username);

// Get a session
echo Session::get('username');

// Delete a session
Session::delete('username');

Session Drivers

Several drivers are provided to enable sessions to be stored in various ways.

// Native session driver
Session::start();

// Cookie driver - sessions are stored in cookies
Session::start('cookie');

// Database driver - sessions are stored in a database (coming soon)
Session::start('database');

Cookies

Handling Cookies

// Set a cookie
Cookie::set('username', $user->username);

// Get a cookie
echo Cookie::get('username');

// Delete a cookie
Cookie::delete('username');

// Currently the encryption key is hardcoded as a property of the Cookie class
// A method to set the key will likely be provided in the future
